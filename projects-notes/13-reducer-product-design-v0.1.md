# Reducer Product Design v0.1

## 目标
重盘 Fuli Plus 中 reducer 的产品角色、更新对象与核心规则，明确：
- reducer 到底更新什么
- 什么不该由 reducer 更新
- 用户反馈如何进入 reducer
- reducer 在多轮闭环中的位置

---

## 一、Reducer 的产品定义
在 Fuli Plus 中，reducer 不是一个“算平均值的公式”，而是：

> 一个把用户反馈转译为 design state 更新的规则层。

它的职责不是直接生成图片，而是根据用户对结果的反馈，决定：
- 哪些设计维度保持不动
- 哪些设计维度应当收束
- 哪些设计维度要反向探索
- 哪些设计维度需要更细粒度展开

换句话说，reducer 是连接“用户反馈”与“下一轮设计状态”的中间机制。

---

## 二、Reducer 不更新什么
### 1. 不直接更新 Physical Product Layer
Physical Product Layer（尺寸、形状、铺设条件等）应作为前置 constraint layer，被较早确认。

默认情况下，用户在生图阶段的喜欢 / 不喜欢，不应直接改动：
- formType
- sizeClass
- room-fit constraints

只有在用户明确指出对象条件不对时，这层才被重新确认。

### 2. 不直接把喜欢 / 不喜欢转成参数平均值
用户反馈先进入证据层，而不是直接进入数值层。

---

## 三、Reducer 的主要更新对象
Reducer 的主体作用于 Pattern Language Layer：
- Color Palette
- Motif
- Arrangement
- Impression
- Style

在后期，也可扩展到 Finishing / Emphasis Layer：
- Rhythm
- Visual Emphasis

---

## 四、Reducer 的输入
Reducer 每一轮至少需要四类输入：

### 1. 当前 base design state
包括每个槽位的：
- current value
- candidate values
- confidence
- state（Open / Narrowed / Locked）
- exploration strategy（wide / narrow / late-lock）

### 2. 本轮变体说明
每张图必须带有明确记录：
- 改了哪些槽位
- 改了哪些轴向
- 改动方向是什么
- 改动幅度多大

### 3. 用户反馈
- 喜欢
- 不喜欢
- 最喜欢 / 最不喜欢
- 可选附加语言说明

### 4. 约束条件
- reference image mode / no reference mode
- brand prior
- product constraints

---

## 五、Reducer 的核心过程

### Step 1：先判层
默认情况下，用户反馈优先进入 Pattern Language Layer。

只有在以下情况，才考虑触发 Physical Product Layer 的重新确认：
- 用户明确说对象形态不对
- 房间 / 尺寸条件变化
- 当前 shape/coverage 约束与图案语言严重冲突

### Step 2：做槽位级归因
根据本轮变体差异与反馈，判断：
- 哪个槽位最可能是主因
- 哪个槽位是次因
- 哪些槽位应保持不动

### Step 3：做轴向级更新
在被选中的槽位内，判断：
- 是哪个轴向更可能需要更新
- 更新方向是什么
- 更新幅度是 wide / narrow / late-lock 中哪种

### Step 4：做状态决策
每个槽位可进入四种更新结果之一：
- 保持（evidence 不足）
- 收束（方向基本正确，缩小范围）
- 反向探索（当前方向错误，但槽位结构仍可用）
- 拓展（当前槽位粒度不足，需要更细子结构）

### Step 5：重组下一轮 base state
新的 base 不是所有结果参数的平均值，而是：
- 由当前证据最强支持的槽位方向重新组成
- 未确定槽位保持开放
- 已收束槽位缩小搜索范围或锁定

---

## 六、Reducer 的证据规则

### 正向证据
当某个槽位/轴向的变化稳定出现在被喜欢的结果中时：
- 增加 positive evidence
- 提高该方向 confidence
- 允许收束或窄范围精修

### 负向证据
当某个槽位/轴向的变化稳定出现在被不喜欢的结果中时：
- 增加 negative evidence
- 排除当前候选方向
- 保持 open 或触发反向探索

### 冲突证据
如果不同结果中的证据互相冲突：
- 暂不收束
- 需要更干净的对照测试
- 或说明当前槽位粒度过粗

---

## 七、Reducer 的四类更新动作

### 1. 保持
条件：
- 证据不足
- 当前轮次无法清楚归因

结果：
- base 不变
- 继续观察或等待下一轮更干净测试

### 2. 收束
条件：
- 当前方向获得稳定正向证据
- 用户偏好在该槽位上开始显现一致性

结果：
- 缩小候选范围
- wide → narrow
- Open → Narrowed / Locked

### 3. 反向探索
条件：
- 当前方向持续获得负向证据
- 但槽位本身仍然是正确控制对象

结果：
- 改变探索方向
- 排除当前方向
- 继续在同一槽位内尝试相反或邻近方向

### 4. 拓展
条件：
- 槽位频繁成为主因
- 但当前轴向结构解释不了用户反馈
- 用户反馈总落在该槽位，但现有参数不够用

结果：
- 在高层槽位下引入更细子参数
- 或拆出新的 bridge / finishing 维度

---

## 八、Reducer 的产品规则

### 规则 1：每轮不测全部槽位
每轮只测试当前最值得测试的 2–3 个槽位。

### 规则 2：每张图都必须有明确变动假设
不是“多出几张图”，而是“做受控实验”。

### 规则 3：反馈先更新证据，不直接更新数值
数值更新是证据积累后的结果，而不是反馈的直接映射。

### 规则 4：高层先收敛，低层后展开
先收敛 Color / Motif / Arrangement / Impression 等高层方向；
后期再引入 Rhythm / Visual Emphasis 或更细子参数。

### 规则 5：Physical Product Layer 默认不进入主 reducer
除非用户明确修改对象条件。

### 规则 6：用户主观感受反馈优先归 Impression
除非有强证据表明问题来自 Style 或 Arrangement 等其他层。

---

## 九、每轮实验规模与图数策略

### 一个更精确的原则
每轮不是按“测几个槽位”来设计，而是按：

> 用户这一轮能够稳定比较多少个设计假设

来设计。

因此，系统不追求一轮测尽可能多，而追求：
- 反馈尽量干净
- 假设尽量可比较
- 用户负担不过重

### MVP 默认策略
- **默认每轮 4 张图**
- **默认主测 1–2 个槽位**
- 不是为了保守，而是为了让反馈可归因

### 1. 诊断轮（diagnostic round）
**目标：** 找当前主因槽位

**建议：**
- 4 张图左右
- 1–2 个主测槽位
- 差异相对明显

**典型结构：**
- 1 张 base
- 2 张单变量偏移
- 1 张联动或边界测试图

### 2. 验证轮（validation round）
**目标：** 验证某个方向是否值得收束

**建议：**
- 3–4 张图
- 1 个主测槽位
- 差异幅度缩小

### 3. 精修轮（refinement round）
**目标：** 在高层方向稳定后做局部微调

**建议：**
- 2–3 张图
- 1 个细槽位或 1 个局部问题
- 差异更小、更局部

### 4. 参考图模式第一轮
**目标：** 在保留参考图 identity 的前提下做局部受控发散

**建议：**
- 4–5 张图
- 包括 1 张 anchor 图
- 其余图只在少量可偏移槽位上做品牌化调整

### 为什么不一轮测更多
如果一轮同时引入过多槽位或过多图：
- 用户认知负担显著增加
- 喜欢/不喜欢反馈更容易变脏
- reducer 难以判断当前主因
- 下一轮反而会更慢

因此，一个核心规则是：

> 不是一轮测尽量多，而是一轮测得尽量干净。

## 十、参考图模式与无参考图模式的 reducer 差异

### 参考图模式
- 初始状态来自 slot prefill from image
- reducer 关注的是：哪些维度保留参考图 identity，哪些维度可向 Fuli 品牌方向偏移
- 第一轮更适合做 anchored local divergence

### 无参考图模式
- 初始状态来自 dialogue elicitation
- reducer 关注的是：用户正在形成中的偏好如何逐步被结构化
- 第一轮更适合做高层方向探索

两种模式初始化不同，但后续都进入“反馈 → 归因 → 状态更新 → 下一轮”的同一闭环。

---

## 十、当前一句话总结
Reducer 不是参数公式，而是：

> 一个在既定对象条件下，把用户偏好反馈逐步转译为设计状态更新规则的产品机制。
