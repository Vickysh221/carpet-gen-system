# Fuli Plus 核心槽位框架 v0.1

## 目标
定义一组足够稳定、足够贴近地毯设计语言的核心槽位，作为系统进行意图解析、变体生成、反馈归因和状态收敛的共同坐标系。

## 槽位设计原则
1. 槽位必须服务于地毯设计，而不是泛化图片美化参数。
2. 槽位需要既能承接用户模糊感受，也能指导后续生成。
3. 槽位之间要尽量减少语义重叠，避免反馈归因失真。
4. 槽位要能支持 Open / Locked 状态更新。
5. 槽位要允许未来长出派生层，但核心骨架先保持稳定。

## 核心槽位建议

### 1. Impression（总体印象 / 气质）
**回答的问题：** 这块地毯整体给人的第一感受是什么？

**承接的用户语言：**
- 温暖 / 治愈 / 安静 / 高级 / 有力量 / 灵动 / 克制

**作用：**
- 作为最上层的总体语义方向
- 决定后续多个槽位的默认范围

**风险：**
- 容易过泛，必须避免直接替代其他所有槽位

---

### 2. Rhythm（节奏 / 疏密 / 动势）
**回答的问题：** 图案内部的节奏感是松还是紧、静还是动、均匀还是跳跃？

**承接的用户语言：**
- 有呼吸感 / 不要太乱 / 更有节奏 / 稳一点 / 跳一点

**作用：**
- 帮助系统把用户对“秩序感”和“动态感”的判断落到可比较维度上

---

### 3. Composition（构成组织）
**回答的问题：** 图案元素是如何被组织起来的？

**承接的用户语言：**
- 不要太满 / 更聚焦 / 更整体 / 更散开 / 更有中心

**作用：**
- 区分用户不喜欢的原因究竟来自布局结构，还是来自色彩/气质

---

### 4. Pattern Language（纹样语言）
**回答的问题：** 图案更接近哪种纹样表达体系？

**承接的用户语言：**
- 几何一点 / 有机一点 / 更像织物 / 不要插画感 / 更抽象

**作用：**
- 保证生成结果不只是“好看的图片”，而更接近地毯/织物图案本身的语言

**与当前 schema 的对应：**
- 可进一步映射为 `Motif` 槽位（geometryDegree / organicDegree / complexity）

---

### 5. Color Atmosphere（色彩气氛）
**回答的问题：** 色彩整体营造的氛围是什么？

**承接的用户语言：**
- 暖一点 / 柔一点 / 不要太灰 / 更沉稳 / 更明亮

**作用：**
- 承接用户最常见的审美反馈
- 作为反馈归因中高频变动槽位

---

### 6. Material Texture Cue（材质/肌理暗示）
**回答的问题：** 视觉上呈现出怎样的织物感、肌理感与触感想象？

**承接的用户语言：**
- 更软 / 更厚 / 更细腻 / 更粗粝 / 更有织感

**作用：**
- 把“地毯感”从纯视觉图像中拉出来
- 防止结果过于平面化、插画化

---

### 7. Contrast Focus（对比与视觉焦点）
**回答的问题：** 视觉焦点和对比强度如何分布？

**承接的用户语言：**
- 不要太跳 / 想有一点亮点 / 更均匀 / 焦点更集中

**作用：**
- 用于判断用户是在拒绝“过强刺激”，还是在嫌弃“缺少记忆点”

## 为什么是这 7 个
这 7 个槽位试图覆盖：
- 总体感受（Impression）
- 动态秩序（Rhythm）
- 结构组织（Composition）
- 图案语言（Pattern Language）
- 色彩氛围（Color Atmosphere）
- 材质想象（Material Texture Cue）
- 焦点控制（Contrast Focus）

它们共同构成一个比“饱和度/色调/滤镜”更贴近地毯设计语言的坐标系。

## 槽位状态建议
每个槽位至少包含：
- current_value
- candidate_values
- confidence
- state（Open / Locked）
- evidence（来自哪些用户反馈）
- next_action（下一轮是探索、微调还是锁定）

## 派生槽位的可能性
在核心槽位稳定后，可以从某些槽位中派生更细分的子槽位。例如：
- Color Atmosphere → 明度 / 冷暖 / 饱和层次
- Rhythm → 疏密 / 重复频率 / 动势方向
- Material Texture Cue → 细腻度 / 厚重感 / 织感强度

但这些派生槽位应当建立在核心槽位之上，而不是替代核心骨架。

## 与 symbiotic being-with 的关系
这个槽位系统并不是替用户决定“正确审美”，而是作为一个共同工作的语义骨架，让用户和系统一起把尚未成形的偏好逐渐拉成可见结构。
