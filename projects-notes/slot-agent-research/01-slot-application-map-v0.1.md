# 《槽位在 agent 产品中的应用地图 v0.1》

## 1. 研究目的
这份文档不聚焦某一个行业竞品，而是尝试从更广泛的 agent 产品方法论出发，研究“槽位（slot）”作为一种结构化中间表示层，在不同类型 agent 系统中如何被使用、它解决了什么问题，以及它位于系统闭环的哪一层。

## 2. 什么是“槽位”
这里的“槽位”不局限于字面上的 slot，也包括 fields、attributes、semantic tags、state schema、template variables、profile dimensions 等类似机制。

在本研究中，槽位被定义为：

> 一种位于用户自然语言/原始输入与系统决策/生成/执行之间的结构化中间表示层。

它通常具备以下特征：
- 将模糊输入拆分为若干相对稳定的语义维度
- 每个维度可以被系统持续引用、更新、比较或沉淀
- 它比自然语言更可控，比底层模型参数更贴近产品语义

## 3. 为什么 agent 需要槽位
槽位不是为了“让结构看起来更整齐”，而是为了缓解 agent 产品中的几个根本矛盾：

### 3.1 模糊输入 vs 稳定执行
用户表达通常是模糊、跳跃、情绪化的；agent 若直接依赖原始表达，很难稳定规划、生成或执行。槽位帮助系统把模糊输入压成较稳定变量。

### 3.2 自然语言灵活 vs 产品可控
自然语言足够灵活，但产品需要有可预期的控制边界。槽位在灵活性与控制性之间提供中层约束。

### 3.3 单轮结果 vs 多轮收敛
如果每轮交互都只基于当前输入重新生成，系统很难真正积累状态。槽位让跨轮连续性和收敛成为可能。

### 3.4 结果直觉化 vs 过程可解释
很多 AI 结果用户可以直觉判断好坏，但系统若无法解释“为什么这样做”，就难以建立信任。槽位为解释提供单位。

## 4. 槽位在 agent 产品中的五种典型应用

### 4.1 需求收敛型槽位
**作用：** 把用户模糊需求压成明确决策变量。

**常见场景：**
- 旅行规划 agent
- 家装/家居 assistant
- 预订/顾问型 agent
- 招聘/匹配类 agent

**典型槽位：**
- 时间
- 预算
- 风格偏好
- 约束条件
- 地点
- 人数
- 目标

**槽位价值：**
用户说不清楚，但系统必须逐步补齐可用于推荐或执行的关键变量。

---

### 4.2 知识映射型槽位
**作用：** 把自然表达映射到知识系统中的结构化对象。

**常见场景：**
- 教育/学习 agent
- 语言辅导 agent
- 知识问答 assistant
- 带 profile 的 RAG 系统

**典型槽位：**
- 知识点类别
- 错误类型
- 难度级别
- 语法点
- 学习状态
- 掌握程度

**槽位价值：**
用户表达本身不能直接成为教学对象；系统需要借助槽位把表达转成可讲解、可练习、可追踪的知识单元。

---

### 4.3 记忆组织型槽位
**作用：** 把历史输入压成可复用的 memory schema。

**常见场景：**
- journaling agent
- reflective agent
- personal assistant
- long-term memory chatbot

**典型槽位：**
- 事件
- 情绪
- 关系对象
- 长期主题
- 未完成事项
- 偏好
- anchor memory

**槽位价值：**
如果记忆只是原文堆积，agent 很难稳定复用。只有经过一定 schema 化，记忆才容易被检索、总结、触发和再次生成。

---

### 4.4 生成控制型槽位
**作用：** 把生成目标拆成若干可控维度，而不是直接 end-to-end 生成。

**常见场景：**
- 图像生成 agent
- 品牌/空间/服装/织物设计系统
- 视频/角色/故事生成系统

**典型槽位：**
- 风格
- 色彩
- 材质
- 构图
- 语气
- 角色设定
- 场景要素

**槽位价值：**
引入中间表示层，提高可控性、可解释性和复用性。

**风险：**
如果停留在静态参数层，槽位就只是“高级控制面板”，而不一定形成 agent 闭环。

---

### 4.5 反馈归因与收敛型槽位
**作用：** 用槽位解释用户反馈到底指向哪个问题维度，并推动下一轮决策。

**常见场景：**
- 创意生成反馈系统
- 多轮推荐/偏好学习系统
- adaptive tutoring
- 多轮设计探索系统

**典型槽位：**
- 当前变化维度
- 当前主因槽位
- 用户偏好证据
- 已锁定维度
- 待探索维度

**槽位价值：**
让“喜欢/不喜欢”“这个更对/那个不对”这样的轻反馈变成系统可使用的诊断信号，从而支持多轮收敛。

## 5. 槽位位于系统的哪一层
分析 agent 产品时，不能只问“有没有槽位”，更要问“槽位位于哪一层”。同一个产品可能同时有多个层级的槽位机制。

### 层 1：输入表单层
槽位只用于收集参数。

### 层 2：生成/执行控制层
槽位用于影响模型输出或行动路径。

### 层 3：状态表示层
槽位作为系统内部连续状态的一部分，被跨轮保留和更新。

### 层 4：反馈解释层
槽位承担归因单位，用于解释用户反馈究竟对应哪个维度。

### 层 5：长期资产层
槽位被沉淀为长期 profile / memory / preference / design state，成为系统可继承资产。

## 6. 如何判断一个产品只是“参数工具”还是更像“slot-based agent”
可以看以下问题：
- 槽位是否会跨轮保留？
- 槽位是否会被系统主动更新？
- 槽位是否参与下一轮决策？
- 槽位是否承担反馈归因作用？
- 槽位是否沉淀为长期状态资产？

如果答案大多是否，那么它更像参数工具；如果答案大多是，则它开始具有 slot-based agent 的特征。

## 7. Fuli Plus 在这张地图中的位置
Fuli Plus 不只是生成控制型槽位系统，它更接近：
- 生成控制型槽位（4.4）
- 反馈归因与收敛型槽位（4.5）

其关键差异在于：
- 用户输入不是直接翻译成 prompt，而是先进入地毯设计语言的槽位系统
- 用户反馈不是简单触发重生成，而是被解释为设计维度信号
- 系统沉淀的不是一次性生成图，而是可复用的 design state

## 8. 当前研究结论
1. 槽位在 agent 产品中，本质上是一种结构化中间表示层。
2. 槽位的价值不只在于控制生成，更在于支持状态连续性、反馈解释和长期资产沉淀。
3. 很多产品拥有“槽位的壳”，但缺少“反馈归因 + 状态更新 + 多轮收敛”的 agent 闭环。
4. Fuli Plus 的价值不在于简单引入槽位，而在于让槽位同时承担语义抽象、反馈解释与设计状态沉淀三种作用。
