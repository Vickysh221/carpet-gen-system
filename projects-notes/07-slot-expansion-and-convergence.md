# 槽位拓展与收束机制 v0.1

## 目标
回答 Fuli Plus 中一个核心机制问题：当系统以 7 个设计槽位组织 design state 时，用户的点赞 / 点踩反馈如何进入闭环，并推动槽位的收束、继续探索或进一步细化。

## 一、两个入口模式

### 模式 A：用户上传参考图
用户带着一个明确参照进入，希望生成结果与参考图相近，但又需要向 Fuli 品牌的整体风格基调靠拢。

#### 机制起点
1. 系统先对参考图进行 7 个核心槽位的预填写（slot prefill）
2. 得到一个初始 reference-based design state
3. 在此基础上生成第一轮候选结果

#### 关键问题
第一轮应该更收敛，还是更发散？

#### 当前判断
第一轮不应完全收敛，也不应完全发散，而应采用：

> **锚定参考图的局部受控发散（anchored local divergence）**

也就是：
- 保留参考图中最强的 identity signal
- 只在少量槽位上做有限偏移
- 每张图都代表“在参考图约束下的一种品牌化调整假设”

#### 为什么不能完全收敛
如果第一轮只是高保真复现参考图，系统几乎无法判断：
- 用户真正希望保留的是哪些维度
- 哪些维度可以向 Fuli 风格靠拢
- “类似”到底是指图案语言类似、色彩类似，还是总体印象类似

#### 为什么不能完全发散
如果第一轮偏移过大，用户会觉得系统背离了上传参考图的初始意图，失去信任。

#### 更稳的第一轮策略
- 1 张 anchor 图：尽量接近参考图的整体槽位状态
- 2–3 张局部偏移图：每次只动 1–2 个槽位，例如 Color Atmosphere / Pattern Language / Material Texture Cue
- 偏移方向优先选择“更接近 Fuli 品牌基调”的调整

#### 关于“是否需要槽位”
即便有参考图，也仍然需要槽位。原因是：
- 图生图风格迁移只能完成视觉相似性迁移
- 但无法显式表示“哪些维度应该保留，哪些维度允许品牌化调整”
- 没有槽位，系统就无法对用户后续反馈做维度归因

因此：
> 图生图 / 风格迁移可以作为生成手段，但不能替代槽位状态层。

---

### 模式 B：用户没有上传参考图
用户通过模糊自然语言进入系统，系统通过问答或选择逐步补充若干槽位值，形成初始 design state，再生成第一轮候选图。

#### 机制起点
1. 用户表达模糊审美意图
2. 系统通过问答补全若干槽位值
3. 形成一组低置信度的初始槽位状态
4. 生成第一轮候选图

#### 第一轮策略
这一轮比参考图模式更需要承担“测试和澄清”的任务，因此应当：
- 保留一个当前最合理的 base 方案
- 生成若干围绕关键槽位变化的对照图
- 让用户反馈具有诊断意义

---

## 二、不要直接做“7 槽位 = 7 张图”
一个重要判断是：

> 第一轮不宜简单地把每个槽位各改一张图，凑成 7 张。

原因：
1. 7 个槽位同时进入用户认知，负担过大
2. 某些槽位在当前轮次并不是主要不确定项
3. 多个低质量单槽位变体会让反馈噪声极高

更合理的方式是：
- 先判断当前最不确定 / 最关键的 2–3 个槽位
- 只围绕这些槽位做受控变体设计
- 其他槽位保持 base

也就是说，系统每一轮测的不是“全部槽位”，而是“当前最值得测的槽位”。

---

## 三、用户反馈不应直接转成“权重平均值”
当前一个危险想法是：
- 用户喜欢某几张图
- 不喜欢某几张图
- 然后把这些图背后的参数做加权平均
- 得到下一轮 base

这个方法的问题在于：
1. 平均值会抹平结构差异
2. 喜欢/不喜欢不是连续数值反馈，而是相对偏好证据
3. 一张图通常包含多个槽位联动变化，直接平均会造成错误归因

因此：
> 用户反馈不应先进入数值平均，而应先进入“证据解释层”。

---

## 四、推荐的闭环机制

### Step 1：维护当前 base design state
每个槽位至少包含：
- current_value
- candidate_values
- confidence
- state（Open / Narrowed / Locked）
- positive_evidence
- negative_evidence
- next_action

### Step 2：选择本轮要测试的槽位
根据以下信号筛选 2–3 个：
- 当前置信度最低
- 与用户目标最相关
- 历史反馈波动最大
- 最可能影响满意度

### Step 3：生成有对照意义的候选图
每张图都必须带有明确的“变动说明”：
- 变了哪些槽位
- 朝哪个方向变
- 变动幅度多大

### Step 4：收集用户反馈
反馈形式可以是：
- 喜欢
- 不喜欢
- 最喜欢 / 最不喜欢
- 可选附加语言说明

### Step 5：做反馈归因
归因的核心不是“哪张图分高”，而是：
- 哪些槽位变化更常出现在被喜欢的图里
- 哪些槽位变化更常出现在被不喜欢的图里
- 哪些组合关系造成了正向或负向影响

### Step 6：更新槽位状态
系统对每个被测试槽位做四类决策之一：
- 保持：证据不足
- 收束：方向大体正确，缩小搜索范围
- 反向探索：当前方向错误，但槽位定义仍够用
- 拓展：当前槽位粒度不足，需要分化出更细子维度

### Step 7：生成下一轮
新的 base 不是简单平均值，而是：
- 从当前最有证据支持的槽位方向重新组成 base
- 未确定槽位保持开放
- 已收束槽位进入窄范围精修或锁定

## 五、点赞 / 点踩如何影响参数
### 结论先行
点赞 / 点踩不宜直接映射为一个数值系数（例如喜欢 = 1，不喜欢 = 0.3）。

更稳的做法是两层：

### 第一层：证据层
每次点赞 / 点踩首先增加某些槽位方向的正向或负向证据。

例如：
- 如果一张图主要改动 Color Atmosphere，并被点赞，则 Color 某方向的 positive evidence 增加
- 如果一张图主要改动 Rhythm，并被点踩，则 Rhythm 某方向的 negative evidence 增加

### 第二层：状态层
当证据累积到一定程度时，才反映到参数：
- 正向证据足够 → 缩窄候选范围 / 提高 confidence
- 负向证据足够 → 排除某候选方向 / 保持 Open
- 证据互相冲突 → 触发更多对照实验
- 证据解释不了反馈 → 触发槽位拓展

因此，点赞 / 点踩影响的不是“最终公式里的一个乘数”，而是一个槽位证据状态。

## 六、什么情况下需要内部 sub 参数变化
当出现以下情况时，可以触发子参数 / 子槽位：
1. 当前槽位长期处于高影响、高不确定状态
2. 用户语言持续落在该槽位，但现有候选值解释力不足
3. 正负反馈在同一槽位下呈现复杂分叉，无法通过单一方向解释
4. 该槽位已经接近收敛，但还需要更细致的局部精修

例如：
- Color Atmosphere 可细化为冷暖、明度、饱和层次
- Rhythm 可细化为疏密、重复频率、动势强弱
- Material Texture Cue 可细化为细腻度、厚重感、织感强度

当前建议是：
> 先在高层槽位上完成大方向收敛，再在接近收敛阶段引入 sub 参数做局部精修。

## 七、关于参考图模式下“相似但品牌化”的控制
如果用户上传参考图，目标通常不是完全原创探索，而是：
- 保留参考图中的关键 identity
- 向 Fuli 品牌调性做有限迁移

这时可以把 7 个槽位分成三组：

### A. 应优先锚定的槽位
例如：
- Pattern Language
- Impression
- Composition

这些更可能构成“为什么它像参考图”的核心。

### B. 可适度品牌化调整的槽位
例如：
- Color Atmosphere
- Material Texture Cue
- Contrast Focus

这些更适合在第一轮做有限偏移。

### C. 需要谨慎测试的槽位
例如：
- Rhythm

某些槽位一旦改动过大，就会让整体感觉立刻偏离原图。

这意味着参考图模式下，系统不是先问“7 个槽位都怎么改”，而是先问：
> 哪些槽位必须保持相似，哪些槽位允许做品牌化偏移。

---

## 八、当前机制判断
1. 系统每一轮测的不是全部 7 个槽位，而是当前最值得测的 2–3 个槽位。
2. 第一轮在参考图模式下应采用“锚定参考图的局部受控发散”。
3. 第一轮在无参考图模式下应采用“基于问答补全后的对照探索”。
4. 用户点赞 / 点踩先进入证据层，而不是直接进入平均值公式。
5. 下一轮 base 应由“证据支持最强的槽位方向”重组，而不是来自所有图参数的简单平均。
6. 子参数 / 子槽位应在高层方向接近收敛后引入，而不是一开始就全部展开。
